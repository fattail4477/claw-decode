# Anthropic 要员工"不要暴露身份" —— Claude Code 51万行泄露源码里的秘密

> 2026年3月31日，Anthropic 因为 npm 打包失误，意外暴露了 Claude Code 的全部源代码。51万行 TypeScript，1900个文件，什么都藏不住了。

---

今天凌晨，安全研究员 Chaofan Shou 发现 Anthropic 的 npm 包里包含了一个 60MB 的 source map 文件，通过它可以完整还原 Claude Code 的 TypeScript 源码。

我花了几个小时把这 51 万行代码过了一遍。不是作为旁观者——而是作为一个正在做 AI Agent 产品的创业者。以下是我发现的最震撼的 10 件事。

---

## 1. 他们在代码里藏了一个虚拟宠物系统

不是开玩笑。Claude Code 的源码里有一个完整的虚拟宠物系统，代号 "Buddy"。

18 种生物：鸭子、鹅、水豚、猫、龙、章鱼、猫头鹰、企鹅、乌龟、蜗牛、幽灵、蝾螈、仙人掌、机器人、兔子、蘑菇……每个都有 ASCII 动画，带帧数切换。

还有稀有度等级：Common → Uncommon → Rare → Epic → Legendary。

**它们可以戴帽子。** 王冠、高帽、螺旋桨帽、光环、巫师帽……

这个功能被一个叫 `BUDDY` 的 feature flag 隐藏了。公开版本的编译过程直接把这段代码删除了。但 Anthropic 内部员工可以用。

## 2. Claude 会"做梦"

源码里有一个叫 "Dream Mode" 的系统。当 Claude Code 空闲时，它会启动一个后台 agent，自动整理你的对话记忆。

四个阶段：

1. **定向**（Orient）—— 查看记忆目录，读索引文件
2. **收集**（Gather）—— 从日志和对话记录里找新信息
3. **整合**（Consolidate）—— 合并新旧信息，把相对日期转成绝对日期，删除矛盾
4. **修剪**（Prune）—— 保持索引在 25KB 以内，删除过时的指针

它的提示词写着：

> "你正在执行一个 dream —— 对你的记忆文件进行反思性整理。将你最近学到的东西综合成持久的、组织良好的记忆，以便未来的 session 能快速定向。"

**它真的在"睡觉时学习"。**

## 3. "不要暴露你的身份"

这是最让我震惊的发现。

源码里有一个 "Undercover Mode"（卧底模式）。当 Anthropic 员工用 Claude Code 向公开的开源仓库提交代码时，这个模式会**自动激活**。

它会从 commit message、PR 标题和 PR 描述中删除所有 Anthropic 内部信息。源代码注释里赫然写着：

> **"你正在公开/开源仓库中以卧底身份行动。你的 commit 信息、PR 标题和 PR 正文不得包含任何 Anthropic 内部信息。不要暴露你的身份。"**

绝对不能泄露的信息包括：
- 内部模型代号（Capybara、Tengu 等动物名字）
- 未发布的模型版本号（opus-4-7、sonnet-4-8）
- 内部仓库或项目名称

**而且这个模式没有强制关闭选项。** 代码注释说："这是为了防止模型代号泄露——如果我们不确定是否在内部仓库中，就保持卧底状态。"

这意味着 Anthropic 员工一直在用 Claude Code 悄悄地向开源项目贡献代码。

## 4. 内部版和公开版天差地别

这是所有 AI 开发者都应该知道的。

**公开版的提示词：**
> "重要：直接切入重点。先试最简单的方法。不要过度。格外简洁。"

**内部版的提示词：**
> "工具调用之间的文本限制在 ≤25 个词。最终回复限制在 ≤100 个词，除非任务需要更多细节。"

就这个差别。公开版用模糊的"简洁"，内部版用精确的数字限制。这就是为什么 Claude Code 用起来感觉那么干脆利落——它真的在数词。

其他差异：

| | 公开版 | 内部版 |
|---|---|---|
| 代码注释 | 没提 | "默认不写注释" |
| 验证 | 无 | 修改3个以上文件必须验证 |
| 功能 | 编译时删除 | 全部启用 |
| 虚假声明 | 基本规则 | "永远不要在输出显示失败时声称所有测试通过" |

## 5. 43 个工具

不是 10 个，不是 20 个。**四十三个**。

每个工具都是独立的、有权限控制的模块：文件读写编辑、Bash 执行、Web 搜索、多 Agent 编排、定时任务调度、远程触发器、LSP 集成、Jupyter 编辑、跨会话消息传递……

还有一个 Sleep 工具，提示词写着"用户可以随时中断睡眠"。

## 6. 记忆系统不用 RAG

这是对很多 AI 创业公司的当头一棒。

Claude Code 的整个记忆系统就是 **markdown 文件 + 一个索引文件**。没有 Pinecone，没有 Weaviate，没有向量嵌入。

复杂之处不在存储，而在维护——上面说的 Dream Mode 就是维护机制。

索引文件必须保持在 25KB 以内。每个条目一行，不超过 150 字符。

**这挑战了整个 RAG 行业的假设。** 也许你不需要向量数据库，你需要的是一个好的记忆整理循环。

## 7. 泄露的模型代号

源码里引用了动物主题的内部代号：

- **Capybara**（水豚）—— 在代码风格注释、宠物系统、模型名混淆中出现
- **Tengu**（天狗）—— 出现在 feature flag 前缀中：`tengu_kairos_cron`、`tengu_hive_evidence`
- **Numbat** —— 在输出效率章节中："当我们发布 numbat 时删除这个章节"

还有未发布的模型版本：`claude-opus-4-6`（知识截止 2025 年 5 月）和 `claude-sonnet-4-6`（知识截止 2025 年 8 月）。

## 8. KAIROS：不等你问就先说话的 Claude

一个还没发布的持久性主动助手模式。Claude 不只是回答——它会**主动发起对话**。

它通过定期的 "tick" 信号保持清醒，监视你的工作，在发现问题时主动发消息，运行定时任务。

源码描述它为"自主代理"，可以根据用户终端是否在前台来调整自主程度。

## 9. 多 Agent 协作网络

Claude Code 可以生成一群 agent，通过共享任务列表协调工作。

更有趣的是跨会话通信：agent 可以通过 Unix domain socket（本地）和 bridge 协议（跨机器）互相发消息。

**这不是一个 CLI 工具——这是一个分布式 agent 网络。**

## 10. 安全团队有"一票否决权"

源码里有一段硬编码的安全指令，注释标注了具体负责人的名字，并写着：

> "重要：未经安全团队审查，不得修改此指令。"

这段指令精确定义了 Claude 在安全相关请求上的边界：可以帮忙做渗透测试和 CTF，不能帮忙做 DoS 和供应链攻击。

---

## 所以呢？

这次泄露揭示了一件事：Claude Code 不是一个 API 包装器。它是一个**完整的 Agent 操作系统** —— 有进程管理、进程间通信、文件系统、定时任务、权限控制。

对于所有做 AI Agent 的人来说，这是一本免费的教科书。而最让我意外的教训是：**简单的东西往往最好用**。Markdown 文件比向量数据库更适合做 Agent 记忆。
